我的第一篇blog about markdown

About markdown

1.关于标题

语法# （一级标题）## （二级标题），以此类推，#号越多，标题级数越小，切记#后需有空格

2.文本基础语法

“>”可表示引用

这是一段引用

星号，一个为斜体，两个为加粗，三个就是斜体加粗

下划线和HTML一样

下划线

还有例如，$可以引用数学符号，“~”“^”可以做上下标(在typora是这样的)

但是这样在GitHub无法正常显示，github上的语法

log2 2,x2

双波浪线可以表示删除线

原神启动

3.列表

有序列表，以“：”来表示

上大学该做的事:

  1. 吃饭
  2. 睡觉
  3. 学习

无序列表，“*”或者“-”

你的人生理想是:

  • 学习
  • 有所作为

空格表，“-[]”

想做的事:

  • 吃饭
  • 睡觉
  • 学习

切记空格

4.代码

三个反单引号

5.表格

以“I”做分割

姓名 住址 电话
ys fafu 233

6.注释

原神^zs

7.横线

以“—”表示

8.链接

必应

“”方括号描述，括号内为网址，括号内可用“”添加注释

“[]”可以引用链接

必应

9.域名和图片

ULR:

https://cn.bing.com/

输入协议可识别

“!”+“[]”+(输入图片地址“注释”)

关于本地图片在GitHub上难以显示的问题，最简单的解决方法是在source目录下建立一个文件夹存放图片，然后在引用就可以显示了（s虽然还是卡）
